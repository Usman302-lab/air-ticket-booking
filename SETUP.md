# SkyBook — Setup & Feature Guide

A full-stack air ticket booking app with AI-powered features built on Node.js, Express, MongoDB, and React.

---

## Prerequisites

| Tool | Version |
|------|---------|
| Node.js | 18+ |
| npm | 9+ |
| MongoDB | Running locally on `mongodb://localhost:27017` |

---

## Quick Start

### 1. Clone and install dependencies

```bash
git clone https://github.com/Usman302-lab/air-ticket-booking.git
cd air-ticket-booking
npm install
```

### 2. Configure environment variables

```bash
cp backend/.env.example backend/.env
```

Open `backend/.env` and fill in the values:

```env
MONGO_URI=mongodb://localhost:27017/air-ticket-booking
JWT_SECRET=replace_with_a_long_random_string
GEMINI_API_KEY=your_google_gemini_api_key
PORT=3001
```

- **`JWT_SECRET`** — any long random string, e.g. run `openssl rand -hex 32` in your terminal
- **`GEMINI_API_KEY`** — get a free key at [aistudio.google.com](https://aistudio.google.com) (Gemini 2.5 Flash is used for all AI features)

### 3. Seed sample data

```bash
node backend/scripts/seed.js
```

This inserts 5 airlines and 15 flights covering routes between Karachi, Lahore, Islamabad, Dubai, Doha, Istanbul, Abu Dhabi, London, Manchester, Kuala Lumpur, and New York.

### 4. Start the app

```bash
npm run dev
```

This starts both servers concurrently:

| Server | URL |
|--------|-----|
| Frontend (Vite + React) | http://localhost:5173 |
| Backend (Express API) | http://localhost:3001 |

## Feature Guide

### Register & Sign In

1. Open http://localhost:5173
2. Click **Get started** to create an account (email + password)
3. Or click **Sign in** if you already have an account
4. After signing in, the navbar reveals: **Flights**, **My Bookings**, and **AI Assistant**

---

### Search Flights

**From the landing page:**
- Enter a departure city (e.g. `Karachi`), arrival city (e.g. `Dubai`), and an optional max price
- Click **Search** — you are taken to the Flights page with results pre-filtered

**From the Flights page (`/flights`):**
- Use the filter bar at the top to refine by departure, arrival, max price, and sort order (price or duration)
- Each flight card shows airline, route, duration, price, and an **AI pricing badge**

---

### AI Pricing Badges

On every flight card, a coloured badge is automatically generated by Gemini 2.5 Flash:

| Badge | Meaning |
|-------|---------|
| 🟢 **Book Now** | Price is below market average — good time to buy |
| 🔵 **Good Deal** | Fair price, no strong reason to wait |
| 🟡 **Wait if Flexible** | Price looks high; worth checking back if your dates are flexible |

Badges appear a moment after the flight list loads (one AI call for all visible flights).

---

### Book a Flight

1. Click **Book Flight** on any flight card (or open the flight detail page)
2. Confirm the booking — the system creates a booking record linked to your account
3. You are redirected to the **Booking Confirmation** page which shows:
   - Booking reference number
   - Flight details (route, date, departure/arrival times, gate)
   - Your boarding pass summary
4. After booking, **AI Travel Recommendations** are shown for your destination:
   - Weather outlook
   - Visa / entry requirements
   - Packing tips
   - Recommended hotel areas

---

### My Bookings (`/my-bookings`)

- Lists all bookings you have made
- Shows flight route, date, status (`booked` / `in process` / `cancelled`)
- Click any booking to view its boarding pass

---

### AI Assistant (`/ai-assistant`)

A chat interface powered by Gemini 2.5 Flash with two modes:

**Natural language flight search**
> "I want to fly from Lahore to London next month under $700"

The AI understands your intent, extracts the search parameters, and automatically navigates to the Flights page with the correct filters applied.

**General travel Q&A**
> "Do I need a visa for Turkey as a Pakistani passport holder?"
> "What's the best time to visit Istanbul?"
> "What should I pack for a winter trip to London?"

The assistant answers directly in the chat panel.

## API Overview

All API routes require a `Bearer <token>` header (obtained from `/auth/signup` or `/auth/signin`).

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/auth/signup` | Register a new user |
| POST | `/auth/signin` | Sign in, returns JWT token |
| GET | `/api/v1/flight` | List / search flights |
| GET | `/api/v1/flight/:flightNumber` | Flight detail |
| POST | `/api/v1/booking` | Create a booking |
| GET | `/api/v1/booking` | Get your bookings |
| GET | `/api/v1/booking/:id` | Booking detail + boarding pass |
| DELETE | `/api/v1/booking/:id` | Cancel a booking |
| POST | `/api/v1/ai/chat` | AI chat / flight search intent |
| POST | `/api/v1/ai/recommendations` | Travel recommendations for a destination |
| POST | `/api/v1/ai/pricing-insight` | Pricing badges for a list of flights |
